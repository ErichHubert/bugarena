# agent/templates

Support material for the global Codex home baseline or optional repo-root overrides. Keep policy files short and use these only when they fit.

- prefer one global home baseline at `~/.codex/AGENTS.md`
- add repo-root `AGENTS.md` only when a cloned repo truly needs repo-specific overrides
- adapt naming, namespaces, and paths to the target repo
- use them as starting points, not blind copy-paste

Included:
- `dotnet-provider-pattern-b.md`
- `capability-broker-v1.md`
- `agent-container-stack.md` for a lean baseline image plus `mise` runtime bootstrap
- `implementation-checklist.md`
