# Implementation Handoff

- Read `references/implementation-checklist.md` before drafting the handoff.
- Report only the commands and validations that actually ran.
- Keep the summary short, concrete, and reviewable.
- Call out docs updates, known gaps, and follow-up explicitly instead of implying success.
- Separate what changed, how it was validated, and what remains risky or deferred.
