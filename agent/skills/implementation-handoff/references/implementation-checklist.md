# Template: Implementation checklist

Use for handoff notes and the PR body.

## Done
- requirement met
- self-reviewed
- validation run, with results
- docs/supporting artifacts updated as needed
- gaps or follow-up called out

## Summary
- what changed
- why

## Architecture
- key boundary or pattern choice
- notable file sizing or class layout decisions

## Validation
- commands actually executed
- result
- gaps

## Docs
- docs updated, or `not needed`

## Risks / Follow-up
- known gaps, limitations, or deferred work

## PR
- meaningful title
- meaningful description
- created with `gh pr create`
